Analise de sistemas

No modulo de analise de sistemas, aprendemos um pouco sobre o ciclo de vida de um projecto e de alguns diagramas, tais como : De classes,de sequência, de colaboração e de componentes. para saber mais, clique aqui

Para concluir este modulo tivemos que aprender sobre alguns diagramas e criar um á nossa escolha, alguns dos diagramas que conheci neste módulo foram :

  • Fluxograma
  • Classses UML
  • Bancos de Dados - Entidade Relacionamento

    • Este Diagrama de entidade-relacionamento de uma base de dados de uma agencia imobiliaria , possui 5 tabelas, Agente Imobiliário, Proprietário, Contrato, Cliente e Imovél.

    No caso da tabela Agente Imobiliário, estes atributos são :

  • Agente ID
  • Nome
  • Morada
  • Número Telefone
  • Email

    • No caso da tabela Proprietário, os atributos são :

  • Proprietário ID
  • Nome
  • Morada
  • Número de Telefone
  • E-Mail

    • No caso da tabela COntrato, os atributos são :

  • Contrato ID
  • Data Inicio
  • Periodo
  • Tipo
  • Cliente ID
  • Imovél ID
  • Agente ID
  • Proprietário ID

    • No caso da tabela Cliente, os atributos são :

  • Cliente ID
  • Nome
  • Morada
  • Numero de Telefone
  • Email

    • No caso da tabela Imovél, os atributos são :

  • Imovél ID
  • Descrição
  • Valor
  • Morada
  • Situação
  • Zona

    • Como podemos observar na imagem, os IDs são Primary Keys na suas próprias tabelas e nas externas, são Foreign Keys, as setas na imagem indicam qual tabela interagem entre si.
